# New-AzDataLakeAnalyticsComputePolicy

## SYNOPSIS
Skapar en principregel för beräkning av Data Lake Analytics för en viss AAD-enhet.

## SYNTAX

```
New-AzDataLakeAnalyticsComputePolicy [-ResourceGroupName <String>] [-Account] <String> [-Name] <String>
 [-ObjectId] <Guid> [-ObjectType] <String> [-MaxAnalyticsUnitsPerJob <Int32>] [-MinPriorityPerJob <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
**New-AzDataLakeAnalyticsComputePolicy** skapar den angivna beräkningsprincipregeln för en viss AAD-enhet i ett Azure Data Lake Analytics-konto.

## EXEMPEL

### Exempel 1: Skapa en beräkningsprincip med bara en regel
```
PS C:\>New-AzDataLakeAnalyticsComputePolicy -Account "contosoadla" -Name "myPolicy" -ObjectId 83cb7ad2-3523-4b82-b909-d478b0d8aea3 -ObjectType User -MaxAnalyticsUnitsPerJob 5
```

Det här kommandot skapar en princip som kallas "myPolicy" i kontot "contosoadla" för användaren med id "83cb7ad2-3523-4b82-b909-d478b0d8aea3" som garanterar att de inte kan skicka något jobb med fler än 5 analysenheter.

### Exempel 2: Skapa en beräkningsprincip med båda regeluppsättningen
```
PS C:\>New-AzDataLakeAnalyticsComputePolicy -Account "contosoadla" -Name "myPolicy" -ObjectId 83cb7ad2-3523-4b82-b909-d478b0d8aea3 -ObjectType User -MaxAnalyticsUnitsPerJob 5 -MinPriorityPerJob 100
```

Det här kommandot skapar en princip som kallas "myPolicy" i kontot "contosoadla" för användaren med id "83cb7ad2-3523-4b82-b909-d478b0d8aea3" som garanterar att de inte kan skicka något jobb med fler än 5 analysenheter eller med en prioritet som är lägre än 100
