# Remove-AzStorageSyncCloudEndpoint

## SYNOPSIS
Det här kommandot tar bort den angivna molnslutpunkten från en synkroniseringsgrupp. Utan minst en molnslutpunkt kan inga andra serverslutpunkter i den här synkroniseringsgruppen synkronisera.

## SYNTAX

### StringParameterSet (standard)
```
Remove-AzStorageSyncCloudEndpoint [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectParameterSet
```
Remove-AzStorageSyncCloudEndpoint [-InputObject] <PSCloudEndpoint> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Remove-AzStorageSyncCloudEndpoint [-ResourceId] <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Det här kommandot tar bort den angivna molnslutpunkten från en synkroniseringsgrupp. Azure-filresursen som molnslutpunktsreferenserna inte ändras av den här processen. Det här kommandot är endast avsett för inaktivering. Att ta bort en molnslutpunkt är en skadligt åtgärd. Serverslutpunkter kan inte synkronisera utan att minst en molnslutpunkt finns. Den här åtgärden bör inte utföras för att lösa synkroniseringsproblem. Om den här Azure-filresursen läggs till i samma synkroniseringsgrupp som en molnslutpunkt kan den leda till konfliktfiler och andra oavsiktliga konsekvenser.

## EXEMPEL

### Exempel 1
```powershell
PS C:\> Remove-AzStorageSyncCloudEndpoint -Force -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -Name "myCloudEndpointName"
```

Med det här kommandot tas molnslutpunkten bort.
