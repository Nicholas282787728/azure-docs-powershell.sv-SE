# New-AzStorageSyncService

## SYNOPSIS
Med det här kommandot skapas en ny tjänst för lagringssynkronisering i en resursgrupp.

## SYNTAX

```
New-AzStorageSyncService [-ResourceGroupName] <String> [-Name] <String> [-Location] <String> [-IncomingTrafficPolicy] <String> [-Tag <Hashtable>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
En lagringssynkroniseringstjänst är den översta resursen för Azure File Sync. Med det här kommandot skapas en ny tjänst för lagringssynkronisering i en resursgrupp. Vi rekommenderar att du skapar så få lagringssynkroniseringstjänster som absolut nödvändigt för att skilja mellan olika grupper av servrar i din organisation. En lagringssynkroniseringstjänst innehåller synkroniseringsgrupper och fungerar även som ett mål för att registrera dina servrar. En server kan endast registreras i en enda lagringssynkroniseringstjänst. Om servrar någon gång behöver delta i synkroniseringen av samma uppsättning filer kan du registrera dem i samma lagringssynkroniseringstjänst.

## EXEMPEL

### Exempel 1
```powershell
PS C:\> New-AzStorageSyncService -ResourceGroupName "myResourceGroup" -Location "myLocation" -StorageSyncServiceName "myStorageSyncServiceName" -IncomingTrafficPolicy "AllowAllTraffic"
```

Det här kommandot skapar en lagringssynkroniseringstjänst.
