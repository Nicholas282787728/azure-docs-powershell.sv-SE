# Restart-AzDeploymentManagerRollout

## SYNOPSIS
Startar om en misslyckad återställning.

## SYNTAX

### Interaktiv (standard)
```
Restart-AzDeploymentManagerRollout [-ResourceGroupName] <String> [-Name] <String> [-SkipSucceeded]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceId
```
Restart-AzDeploymentManagerRollout [-ResourceId] <String> [-SkipSucceeded]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject
```
Restart-AzDeploymentManagerRollout [-InputObject] <PSRollout> [-SkipSucceeded]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Restart-AzDeploymentManagerRollout** startar om en misslyckad rollout och returnerar ett objekt som representerar den utrullningen med all detaljerad information om utrullningen.
Ange rollouten efter dess namn och resursgruppnamn. Alternativt kan du ange rollout-objektet eller Resurs-ID:t.
Med SkipSucceeded (valfri parameter) kan du hoppa över alla efterföljande steg i föregående körning av utrullningen.

## EXEMPEL

### Exempel 1
```powershell
PS C:\> Restart-AzDeploymentManagerRollout -ResourceGroupName ContosoResourceGroup -Name ContosoRollout -SkipSucceeded
```

Det här kommandot startar om en rollout med namnet ContosoRollout i ContosoResourceGroup. Flaggan SkipSucceeded anger att alla steg som redan har körts bör hoppas över och att utrullningen ska fortsätta köras från den senaste misslyckades.

### Exempel 2: Starta om en utrullning med resursidentifieraren
```powershell
PS C:\> Restart-AzDeploymentManagerRollout -ResourceId "/subscriptions/subscriptionId/resourcegroups/ContosoResourceGroup/providers/Microsoft.DeploymentManager/rollouts/ContosoRollout"
```

Det här kommandot startar om en rollout med namnet ContosoRollout i ContosoResourceGroup.

### Exempel 3: Starta om en utrullning med hjälp av rollout-objektet.
```powershell
PS C:\> Get-AzDeploymentManagerRollout -InputObject $rolloutObject
```

Det här kommandot startar om en rollout vars namn och ResourceGroup matchar egenskaperna Namn och ResourceGroupName för $rolloutObject respektive.
