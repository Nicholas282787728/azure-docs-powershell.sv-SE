# Restore-AzKeyVaultManagedStorageAccount

## SYNOPSIS
Återställer ett hanterat lagringskonto i ett nyckelvalv från en säkerhetskopia.

## SYNTAX

### ByVaultName (standard)
```
Restore-AzKeyVaultManagedStorageAccount [-VaultName] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Restore-AzKeyVaultManagedStorageAccount [-InputObject] <PSKeyVault> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Restore-AzKeyVaultManagedStorageAccount [-ResourceId] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Restore-AzKeyVaultManagedStorageAccount** skapar ett hanterat lagringskonto i det angivna nyckelvalvet från en säkerhetskopia.
Det här hanterade lagringskontot är en kopia av det säkerhetskopierade hanterade lagringskontot i indatafilen och har samma namn som originalet.
Om nyckelvalvet redan innehåller ett hanterat lagringskonto med samma namn misslyckas den här cmdleten i stället för att skriva över originalet.
Nyckelvalvet som du återställer kontot för hanterad lagring till kan vara ett annat än nyckelvalvet som du säkerhetskopierade kontot för hanterad lagring från.
Men nyckelvalvet måste använda samma prenumeration och vara i en Azure-region i samma geografi (till exempel Nordamerika).
Se Microsoft Azure Trust Center https://azure.microsoft.com/support/trust-center/) (för mappning av Azure-regioner till geografiska områden).

## EXEMPEL

### Exempel 1: Återställa ett säkerhetskopierat hanterat lagringskonto
```powershell
PS C:\> Restore-AzKeyVaultManagedStorageAccount -VaultName 'MyKeyVault' -InputFile "C:\Backup.blob"
```
