# Undo-AzRecoveryServicesBackupItemDeletion

## SYNOPSIS
Om ett säkerhetskopianobjekt tas bort och finns i ett mjukt borttagna tillstånd innebär det här kommandot att objektet åter visas i ett läge där data bevaras för alltid 

## SYNTAX

```
Undo-AzRecoveryServicesBackupItemDeletion [-Item] <ItemBase> [-Force] [-VaultId <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Med Undo-AzRecoveryServicesBackupItemDeletion cmdleten återställs ett mjukt borttaget objekt till ett läge där skyddet stoppas, men data bevaras för alltid.

## EXEMPEL

### Exempel 1
```powershell
PS C:\> $Cont = Get-AzRecoveryServicesBackupContainer -ContainerType AzureVM
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM 
PS C:\> Disable-AzRecoveryServicesBackupProtection -Item $PI[0] -RemoveRecoveryPoints
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM | Where-Object {$_.DeleteState -eq "ToBeDeleted"}
PS C:\> Undo-AzRecoveryServicesBackupItemDeletion -Item $PI[0]
```

Det första kommandot får en matris med behållare för säkerhetskopiering och lagrar den sedan $Cont matrisen.
Det andra kommandot hämtar säkerhetskopian som motsvarar det första behållarobjektet och lagrar det sedan i den $PI variabeln.
Det tredje kommandot inaktiverar säkerhetskopieringsskydd för objektet i $PI 0 och placerar objektet i \[ \] ett mjukdelat tillstånd.
Det fjärde kommandot får objektet i ett mjukdelat tillstånd.
Det senaste kommandot tar den mjukändade virtuella maskinerna till ett tillstånd där skyddet stoppas, men data bevaras för alltid.
