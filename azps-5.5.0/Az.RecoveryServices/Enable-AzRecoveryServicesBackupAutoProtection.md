# Enable-AzRecoveryServicesBackupAutoProtection

## SYNOPSIS
Med **cmdleten Enable-AzRecoveryServicesBackupAutoProtection** konfigureras automatiskt skydd av aktuella och framtida SQL-DBs inom den angivna instansen med den medföljande principen.

## SYNTAX

```
Enable-AzRecoveryServicesBackupAutoProtection [-InputItem] <ProtectableItemBase>
 [-BackupManagementType] <BackupManagementType> [-WorkloadType] <WorkloadType> [-Policy] <PolicyBase>
 [-PassThru] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESKRIVNING
Med det här kommandot kan användare automatiskt skydda alla befintliga oskyddade SQL-db-databaser och db-databaser som läggs till senare med den givna principen.  Eftersom instruktionen är att säkerhetskopiera alla framtida DBs görs åtgärden på SQLInstance-nivå, och azure-säkerhetskopieringstjänsten söker sedan regelbundet igenom automatiskt skyddade behållare efter nya DBs och skyddar dem automatiskt.

## EXEMPEL

### Exempel 1
```powershell
PS C:\> $Pol = Get-AzRecoveryServicesBackupProtectionPolicy -Name "DefaultSQLPolicy"
PS C:\> $SQLInstance = Get-AzRecoveryServicesBackupProtectableItem -workloadType MSSQL -ItemType SQLInstance -VaultId $targetVault.ID -Name "MSSQLInstance" -ServerName "TestSQLServer" 
PS C:\> Enable-AzRecoveryServicesBackupAutoProtection -InputItem $SQLInstance -BackupManagementType AzureWorkload -WorkloadType MSSQL -Policy $Pol -VaultId $targetvault.ID 
```

Den första cmdleten får ett standardprincipobjekt och lagrar den sedan $Pol variabeln.
Den andra cmdleten hämtar relevant SQLInstance som är ett skyddbart objekt. Med det tredje kommandot konfigureras sedan automatiskt skydd för den här instansen med hjälp av principen i $Pol.

### Exempel 2

Med de här kommandona kan användare automatiskt skydda alla befintliga oskyddade DB-system och db-data som läggs till senare med den givna principen. (autogenererat)

```powershell <!-- Aladdin Generated Example --> 
Enable-AzRecoveryServicesBackupAutoProtection -BackupManagementType AzureVM -InputItem <ProtectableItemBase> -Policy $Pol -VaultId $vault.ID -WorkloadType AzureVM
```
