# Add-AzWebAppAccessRestrictionRule

## SYNOPSIS
Lägger till en Restiction-regel för Access i en Azure Web App.

## SYNTAX

### IpAddressParameterSet (standard)
```
Add-AzWebAppAccessRestrictionRule [-ResourceGroupName] <String> [-WebAppName] <String> [-Name <String>]
 [-Description <String>] -Priority <UInt32> [-Action <String>] [-SlotName <String>] [-TargetScmSite]
 -IpAddress <String> [-PassThru] [-HttpHeader <Hashtable>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Add-AzWebAppAccessRestrictionRule** lägger till en åtkomstbegränsningsregel i en Azure Web App.

## EXEMPEL

### Exempel 1: Lägga till en regel för åtkomstbegränsning för IpAddress i en webbapp
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name IpRule -Priority 200 -Action Allow -IpAddress 10.10.0.0/8
```

Det här kommandot lägger till en åtkomstbegränsningsregel med prioritet 200 och ip-intervall till en webbapp med namnet ContosoSite som tillhör resursgruppen Default-Web-WestUS.

### Exempel 2: Lägga till en begränsningsregel för slutpunkt för åtkomstbegränsning för undernät i en webbapp
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name SubnetRule -Priority 300 -Action Allow -SubnetName appgw-subnet -VirtualNetworkName corp-vnet
```

Med det här kommandot läggs en åtkomstbegränsningsregel med prioritet 300 och med undernätets appgw-undernät i corp-vnet till en Web App med namnet ContosoSite som tillhör resursgruppen Default-Web-WestUS.

### Exempel 3: Lägga till en servicetag-begränsningsregel för åtkomst i en webbapp
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name ServiceTagRule -Priority 200 -Action Allow -ServiceTag AzureFrontDoor.Backend
```

Det här kommandot lägger till en åtkomstbegränsningsregel med prioritet 200 och en tjänsttagg som representerar Azure Front Door IP-omfattning till en webbapp med namnet ContosoSite som tillhör resursgruppen Default-Web-WestUS.

### Exempel 4: Lägga till en regel för åtkomstbegränsning med flera adresser i en webbapp
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name MultipleIpRule -Priority 200 -Action Allow -IpAddress "10.10.0.0/8,192.168.0.0/16"
```

Det här kommandot lägger till en åtkomstbegränsningsregel med prioritet 200 och två ip-intervall till en webbapp med namnet ContosoSite som tillhör resursgruppen Default-Web-WestUS.

### Exempel 5: Lägga till en begränsningsregel för åtkomst med http-sidhuvud i en webbapp
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name MultipleIpRule -Priority 400 -Action Allow -ServiceTag AzureFrontDoor.Backend
-HttpHeader @{'x-forwarded-host' = 'www.contoso.com', 'app.contoso.com'; 'x-azure-fdid' = '355deb06-47c4-4ba4-9641-c7d7a98b913e'}
```

Det här kommandot lägger till en åtkomstbegränsningsregel med prioritet 400 för Service Tag AzureFrontDoor.Backend och begränsar ytterligare åtkomst endast till http-rubriker med vissa värden till en webbapp med namnet ContosoSite som tillhör resursgruppen Default-Web-WestUS.
