# Remove-AzIntegrationAccountReceivedIcn

## SYNOPSIS
Denna cmdlet tar bort ett specifikt mottaget överföringskontrollnummer per avtal och kontrollnummervärde.

## SYNTAX

```
Remove-AzIntegrationAccountReceivedIcn -ResourceGroupName <String> -Name <String> -AgreementName <String>
 -ControlNumberValue <String> [-AgreementType <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Den här cmdleten är avsedd att användas för katastrofåterställningsscenarier för att ta bort ett mottaget överföringskontrollnummer från integrationskontot så att B2B-kopplingen kan bearbeta igen meddelandet när dubblettnummeridentifiering har aktiverats.
I sällsynta fall kan det mottagna överföringskontrollnumret reserveras kort före en katastrof och innan B2B-kopplingen avvisar överföringsnumret som felaktigt.
I sådana fall kanske åtgärden vill aktivera återställningswebbplatsen för att bearbeta igen samma överföring när dess nyttolast korrigerats.
Ange parametern "-AgreementType" för att ange om X12- eller Edifact-kontrollnummer ska returneras
