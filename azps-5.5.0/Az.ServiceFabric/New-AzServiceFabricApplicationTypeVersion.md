# New-AzServiceFabricApplicationTypeVersion

## SYNOPSIS
Skapa en ny programtypsversion under angiven resursgrupp och angivet resurskluster.

## SYNTAX

```
New-AzServiceFabricApplicationTypeVersion [-ResourceGroupName] <String> [-ClusterName] <String>
 [-Name] <String> [-Version] <String> -PackageUrl <String> [-DefaultParameter <Hashtable>] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Denna cmdlet skapar en ny programtypsversion med det paket som anges i -PackageUrl. Detta bör vara tillgängligt via en REST-slutpunkt som Azure Resource Manager använder under distributionen och innehöll programmet paketerat och zippat med filnamnstillägget .sfpkg. Det här kommandot skapar programtypen om den inte redan finns.

## EXEMPEL

### Exempel 1
```powershell
PS C:\> $resourceGroupName = "testRG"
PS C:\> $clusterName = "testCluster"
PS C:\> $appTypeName = "testAppType"
PS C:\> $version = "v1"
PS C:\> $packageUrl = "https://sftestapp.blob.core.windows.net/sftestapp/testApp_1.0.sfpkg"
PS C:\> New-AzServiceFabricApplicationTypeVersion -ResourceGroupName $resourceGroupName -ClusterName $clusterName -Name $appTypeName -Version $version -PackageUrl $packageUrl -Verbose
```

Det här exemplet skapar en programtypsversion "v1" under typen "testAppType". Versionen i programmanifestet som finns i paketet ska ha samma version som den som anges i -Version.
