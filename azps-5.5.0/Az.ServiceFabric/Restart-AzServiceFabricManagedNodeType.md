# Restart-AzServiceFabricManagedNodeType

## SYNOPSIS
Starta om specifika noder från nodtypen.

## SYNTAX

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Starta om specifika noder från nodtypen. Tjänstens strukturnoder inaktiveras innan du startar om de virtuella maskinerna och aktiverar dem igen när de har återaktiverats. Om detta görs för primära nodtyper kan det ta ett tag eftersom det kanske inte startar om alla noder på samma gång. Använd -ForceRestart tvinga åtgärden även om det inte går att inaktivera noder på tjänsten, men använd med försiktighet eftersom det kan leda till dataförlust om tillståndsfulla arbetsbelastningar körs på noden.

## EXEMPEL

### Exempel 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Starta om noden 0 och 3 i nodtypen.
