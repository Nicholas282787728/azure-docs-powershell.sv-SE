# Start-AzDataFactoryV2DataFlowDebugSession

## SYNOPSIS
Startar en felsökningssession för dataflöde i Azure Data Factory

## SYNTAX

### ByFactoryName (standard)
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-ResourceGroupName] <String> [-DataFactoryName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByResourceId
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob] [-ResourceId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Det här långvariga kommandot startar en dataflödesfelsökningssession för kommande felsökningskommandon. Det här kommandot kan bifogas med en körningsdefinition för integrering för att konfigurera storlek/typ för felsökningssessionskluster.
PowerShell-kommandosekvensen för felsökning av dataflöde bör vara:
1. Start-AzDataFactoryV2DataFlowDebugSession
1. Add-AzDataFactoryV2DataFlowDebugSessionPackage
1. Invoke-AzDataFactoryV2DataFlowDebugSessionCommand (upprepa det här steget för olika kommandon/mål eller upprepa steg 2-3 för att ändra paketfilen)
1. Stop-AzDataFactoryV2DataFlowDebugSession
