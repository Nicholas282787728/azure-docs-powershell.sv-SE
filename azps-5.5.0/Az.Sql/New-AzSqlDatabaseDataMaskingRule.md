# New-AzSqlDatabaseDataMaskingRule

## SYNOPSIS
Skapar en datamaskeringsregel för en databas.

## SYNTAX

```
New-AzSqlDatabaseDataMaskingRule -MaskingFunction <String> [-PrefixSize <UInt32>] [-ReplacementString <String>]
 [-SuffixSize <UInt32>] [-NumberFrom <Double>] [-NumberTo <Double>] [-PassThru] -SchemaName <String>
 -TableName <String> -ColumnName <String> [-ServerName] <String> [-DatabaseName] <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESKRIVNING
**Cmdleten New-AzSqlDatabaseDataMaskingRule** skapar en datamaskeringsregel för en Azure SQL-databas.
Använd parametrarna *ResourceGroupName, ServerName* och *DatabaseName* för att identifiera regeln om du vill använda cmdleten.
Ange *Tabellnamn och* *Kolumnnamn* för att ange målet för regeln och *parametern MaskingFunction* för att definiera hur data maskeras.
Om *MaskingFunction* har värdet Tal eller Text kan du ange parametrarna *NumberFrom* och *NumberTo* för talmaskering eller *PrefixSize,* *ReplacementString* och *SuffixSize* för textmaskering.
Om kommandot lyckas och parametern *PassThru* används returnerar cmdleten ett objekt som beskriver datamaskregelns egenskaper utöver regelidentifierarna.
Regelidentifierare omfattar, men är inte begränsade till, *ResourceGroupName,* *ServerName,* *DatabaseName* och *RuleID.*
Den här cmdleten stöds också av tjänsten Stretcha databas för SQL Server i Azure.

## EXEMPEL

### Exempel 1: Skapa en datamaskregel för en talkolumn i en databas
```
PS C:\>New-AzSqlDatabaseDataMaskingRule -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"  -SchemaName "Schema01" -TableName "Table01" -ColumnName "Column01" -MaskingFunction "Number" -NumberFrom 5 -NumberTo 14
```

Med det här kommandot skapas en datamaskeringsregel för kolumnen Kolumn01 i tabellen Tabell01 i schemat schema01.
Databasen Databas01 innehåller alla dessa objekt.
Regeln är en talmaskregel som använder ett slumptal mellan 5 och 14 som maskvärde.
