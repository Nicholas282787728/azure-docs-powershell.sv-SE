# Set-AzSqlDatabaseDataMaskingPolicy

## SYNOPSIS
Anger datamasker för en databas.

## SYNTAX

```
Set-AzSqlDatabaseDataMaskingPolicy [-PassThru] [-PrivilegedUsers <String>] [-DataMaskingState <String>]
 [-ServerName] <String> [-DatabaseName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Set-AzSqlDatabaseDataMaskingPolicy** anger datamaskeringsprincipen för en Azure SQL-databas.
Använd parametrarna *ResourceGroupName, ServerName* och *DatabaseName* för att identifiera databasen om du vill använda den här cmdleten.
Du kan ange *parametern DataMaskingState* för att ange om datamaskeringsåtgärder ska aktiveras eller inaktiveras.
Om cmdleten lyckas och *PassThru-parametern* används returneras ett objekt som beskriver den aktuella datamaskeringsprincipen utöver databasidentifierarna.
Databasidentifierare omfattar, men är inte begränsade till, **ResourceGroupName,** **ServerName** och **DatabaseName.**
Den här cmdleten stöds också av tjänsten Stretcha databas för SQL Server i Azure.

## EXEMPEL

### Exempel 1: Ange datamaskeringsprincip för en databas
```
PS C:\>Set-AzSqlDatabaseDataMaskingPolicy -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01 -PrivilegedUsers "public" -DataMaskingState "Enabled"
```

Det här kommandot anger datamaskeringsprincipen för en databas med namnet databas01 på servern server01.
