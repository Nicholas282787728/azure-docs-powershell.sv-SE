# New-AzSqlServerFirewallRule

## SYNOPSIS
Skapar en brandväggsregel för en SQL Database-server.

## SYNTAX

### UserSpecifiedRuleSet
```
New-AzSqlServerFirewallRule -FirewallRuleName <String> -StartIpAddress <String> -EndIpAddress <String>
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### AzureIpruleSet
```
New-AzSqlServerFirewallRule [-AllowAllAzureIPs] [-ServerName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **New-AzSqlServerFirewallRule** skapar en brandväggsregel för den angivna Azure SQL Database-servern.

## EXEMPEL

### Exempel 1: Skapa en brandväggsregel
```
PS C:\>New-AzSqlServerFirewallRule -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -FirewallRuleName "Rule01" -StartIpAddress "192.168.0.198" -EndIpAddress "192.168.0.199"
ResourceGroupName : ResourceGroup01
ServerName        : Server01
StartIpAddress    : 192.168.0.198
EndIpAddress      : 192.168.0.199
FirewallRuleName  : Rule01
```

Det här kommandot skapar en brandväggsregel med namnet Rule01 på servern Server01.
Regeln innehåller de angivna start- och slut-IP-adresserna.

### Exempel 2: Skapa en brandväggsregel som gör att alla Azure IP-adresser får åtkomst till servern
```
PS C:\>New-AzSqlServerFirewallRule -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -AllowAllAzureIPs
```

Det här kommandot skapar en brandväggsregel på servern med namnet Server01 som tillhör resursgruppen ResourceGroup01.
Eftersom *parametern AllowAllAzureIPs* används ger brandväggsregeln alla Azure IP-adresser åtkomst till servern.

## PARAMETERS

### -AllowAllAzureIPs
Anger att den här brandväggsregeln tillåter alla Azure IP-adresser att få åtkomst till servern.
Du kan inte använda den här parametern om du tänker använda parametrarna *FirewallRuleName,* *StartIpAddress* och *EndIpAddress.*
Om du vill tillåta att Azure-IP:er får åtkomst till servern bör den här parametern användas i ett separat cmdlet-anrop som inte använder parametrarna *FirewallRuleName,* *StartIpAddress* och *EndIpAddress.*
