# Update-AzSqlServerAdvancedThreatProtectionSetting

## SYNOPSIS
Anger en avancerad inställning för skydd mot hot på en server.

## SYNTAX

```
Update-AzSqlServerAdvancedThreatProtectionSetting [-PassThru] [-NotificationRecipientsEmails <String>]
 [-EmailAdmins <Boolean>] [-ExcludedDetectionType <String[]>] [-StorageAccountName <String>]
 [-RetentionInDays <UInt32>] -ServerName <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
**Cmdleten Update-AzSqlServerAdvancedThreatProtectionSetting** ställer in en avancerad inställning för skydd mot hot på en Azure SQL-server.
För att kunna aktivera avancerat skydd på en server måste granskningsinställningarna vara aktiverade på servern.
Om du vill använda den här cmdleten anger du *parametrarna ResourceGroupName* och ServerName för att identifiera servern.

## EXEMPEL

### Exempel 1: Ange avancerade inställningar för skydd mot hot för en databas
```powershell
PS C:\>Update-AzSqlServerAdvancedThreatProtectionSetting -ResourceGroupName "ResourceGroup11" -ServerName "Server01" -NotificationRecipientsEmails "admin01@contoso.com;secadmin@contoso.com" -EmailAdmins $False -ExcludedDetectionType "Sql_Injection_Vulnerability","SQL_Injection" -StorageAccountName "mystorageAccount"
```

Det här kommandot anger avancerade inställningar för skydd mot hot för en server med namnet Server01.

### Exempel 2

Anger en avancerad inställning för skydd mot hot på en server. (autogenererat)

```powershell
<!-- Aladdin Generated Example -->
Update-AzSqlServerAdvancedThreatProtectionSetting -EmailAdmins $false -ResourceGroupName 'ResourceGroup11' -RetentionInDays <UInt32> -ServerName 'Server01' -StorageAccountName 'mystorageAccount'
```
