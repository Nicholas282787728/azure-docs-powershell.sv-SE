# Add-AzRouteConfig

## SYNOPSIS
Lägger till en route i en routetabell.

## SYNTAX

```
Add-AzRouteConfig -RouteTable <PSRouteTable> [-Name <String>] [-AddressPrefix <String>] [-NextHopType <String>]
 [-NextHopIpAddress <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Add-AzRouteConfig** lägger till en route till en Azure-routetabell.

## EXEMPEL

### Exempel 1: Lägga till en route i en routetabell
```powershell
PS C:\>$RouteTable = Get-AzRouteTable -ResourceGroupName "ResourceGroup11" -Name "RouteTable01"
PS C:\> Add-AzRouteConfig -Name "Route13" -AddressPrefix 10.3.0.0/16 -NextHopType "VnetLocal" -RouteTable $RouteTable
```

Det första kommandot får en routetabell med namnet RouteTable01 med hjälp av Get-AzRouteTable-cmdleten.
Kommandot lagrar tabellen i den $RouteTable variabeln.
Det andra kommandot lägger till en route med namnet Route13 i tabellen som lagras i $RouteTable.
Denna route vidarebefordrar paket till det lokala virtuella nätverket.

### -NextHopType
Anger hur denna route vidarebefordrar paket.
De godtagbara värdena för den här parametern är:
- Internet.
Standardgatewayen för Internet som tillhandahålls av Azure. 
- Inget.
Om du anger det här värdet vidarebefordrar inte vägen paket. 
- VirtualAppliance.
En virtuell enhet som du lägger till i ditt virtuella Azure-nätverk. 
- VirtualNetworkGateway.
En virtuell privat nätverksgateway i Azure Server till server. 
- VnetLocal.
Det lokala virtuella nätverket.
Om du har två undernät, 10.1.0.0/16 och 10.2.0.0/16 i samma virtuella nätverk, markerar du värdet VnetLocal för varje undernät som ska vidarebefordras till det andra undernätet.
