# New-AzRouteConfig

## SYNOPSIS
Skapar en route för en routetabell.

## SYNTAX

```
New-AzRouteConfig [-Name <String>] [-AddressPrefix <String>] [-NextHopType <String>]
 [-NextHopIpAddress <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **New-AzRouteConfig** skapar en route för en Azure-routetabell.

## EXEMPEL

### Exempel 1: Skapa en route
```powershell
PS C:\>$Route = New-AzRouteConfig -Name "Route07" -AddressPrefix 10.1.0.0/16 -NextHopType "VnetLocal"
PS C:\> $Route
Name              : Route07
Id                : 
Etag              : 
ProvisioningState : 
AddressPrefix     : 10.1.0.0/16
NextHopType       : VnetLocal
NextHopIpAddress  :
```

Det första kommandot skapar en route med namnet Route07 och lagrar den sedan $Route variabeln.
Denna route vidarebefordrar paket till det lokala virtuella nätverket.
Det andra kommandot visar egenskaperna för sträckan.

### Exempel 2

Skapar en route för en routetabell. (autogenererat)

<!-- Aladdin Generated Example -->
```powershell
New-AzRouteConfig -AddressPrefix 10.1.0.0/16 -Name 'Route07' -NextHopIpAddress '12.0.0.5' -NextHopType 'VnetLocal'
```

### -NextHopType
Anger hur denna route vidarebefordrar paket.
De godtagbara värdena för den här parametern är:
- Internet.
Standardgatewayen för Internet som tillhandahålls av Azure. 
- Inget.
Om du anger det här värdet vidarebefordrar inte vägen paket. 
- VirtualAppliance.
En virtuell enhet som du lägger till i ditt virtuella Azure-nätverk. 
- VirtualNetworkGateway.
En virtuell privat nätverksgateway i Azure Server till server. 
- VnetLocal.
Det lokala virtuella nätverket.
Om du har två undernät, 10.1.0.0/16 och 10.2.0.0/16 i samma virtuella nätverk, väljer du ett värde för VnetLocal för varje undernät som ska vidarebefordras till det andra undernätet.
