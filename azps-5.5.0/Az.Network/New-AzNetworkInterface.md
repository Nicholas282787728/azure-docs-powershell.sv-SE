## BESKRIVNING
Cmdleten **New-AzNetworkInterface** skapar ett Azure-nätverksgränssnitt.

## EXEMPEL

### Exempel 1: Skapa ett Azure-nätverksgränssnitt
```powershell
PS C:\>New-AzNetworkInterface -Name "NetworkInterface1" -ResourceGroupName "ResourceGroup1" -Location "centralus" -SubnetId "/subscriptions/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/resourceGroups/ResourceGroup1/providers/Microsoft.Network/virtualNetworks/VirtualNetwork1/subnets/Subnet1" -IpConfigurationName "IPConfiguration1" -DnsServer "8.8.8.8", "8.8.4.4"
```

Det här kommandot skapar ett nätverksgränssnitt med namnet NetworkInterface001 med en dynamiskt tilldelad privat IP-adress från Undernät1 i det virtuella nätverket VirtualNetwork1. Kommandot tilldelar också två DNS-servrar till nätverksgränssnittet. Den underordnade IPConfiguration-resursen skapas automatiskt med namnet IPConfiguration1.

### Exempel 2: Skapa ett Azure-nätverksgränssnitt med ett IP-konfigurationsobjekt
```powershell
PS C:\>$Subnet = Get-AzVirtualNetwork -Name "VirtualNetwork1" -ResourceGroupName "ResourceGroup1" 
PS C:\>$IPconfig = New-AzNetworkInterfaceIpConfig -Name "IPConfig1" -PrivateIpAddressVersion IPv4 -PrivateIpAddress "10.0.1.10" -SubnetId $Subnet.Subnets[0].Id
PS C:\> New-AzNetworkInterface -Name "NetworkInterface1" -ResourceGroupName "ResourceGroup1" -Location "centralus" -IpConfiguration $IPconfig
```

I det här exemplet skapas ett nytt nätverksgränssnitt med ett IP-konfigurationsobjekt. IP-konfigurationsobjektet anger en statisk privat IPv4-adress.
Med det första kommandot hämtas ett befintligt angivet virtuellt nätverk som används för att tilldela undernätet i det andra kommandot.
Det andra kommandot skapar en IP-konfiguration för nätverksgränssnittet med namnet IPConfig1 och lagrar konfigurationen i variabeln $IPconfig.
Det tredje kommandot skapar ett nätverksgränssnitt med namnet NetworkInterface1 som använder ip-konfigurationen för nätverksgränssnittet som lagras i variabeln $IPconfig.
