# Repair-AzVmssServiceFabricUpdateDomain

## SYNOPSIS
Domängenväg för manuell plattformsuppdatering för att uppdatera virtuella maskiner i en skalskala för servicematerial.

## SYNTAX

### DefaultParameter (standard)
```
Repair-AzVmssServiceFabricUpdateDomain [-ResourceGroupName] <String> [-VMScaleSetName] <String>
 [-PlatformUpdateDomain] <Int32> [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ResourceIdParameter
```
Repair-AzVmssServiceFabricUpdateDomain [-PlatformUpdateDomain] <Int32> [-ResourceId] <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ObjectParameter
```
Repair-AzVmssServiceFabricUpdateDomain [-PlatformUpdateDomain] <Int32>
 [-VirtualMachineScaleSet] <PSVirtualMachineScaleSet> [-AsJob] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Tvinga domänuppdatering av manuell plattform för att uppdatera virtuella datorer i en skalskala för servicematerial.

## EXEMPEL

### Exempel 1
```
PS C:\> Repair-AzVmssServiceFabricUpdateDomain -ResourceGroupName $rgname -VMScaleSetName $vmssName -PlatformUpdateDomain 0
```

Det här kommandot tvingar uppdateringsordningen för tjänsten att gå på UD 0 för den virtuella datorskalauppsättning som anges av resursgruppnamnet och skaluppsättningsnamnet.

### Exempel 2
```
PS C:\> $vmss = Get-AzVmss -ResourceGroupName $rgname -VMScaleSetName $vmssName
PS C:\> Repair-AzVmssServiceFabricUpdateDomain -VirtualMachineScaleSet $vmss -PlatformUpdateDomain 1
```

Det här kommandot tvingar uppdateringsordningen för tjänsten på UD 1 för den virtuella datorskalauppsättning som anges av VM-skalningsuppsättningsobjekt.

### Exempel 3
```
PS C:\> Repair-AzVmssServiceFabricUpdateDomain -ResourceId $resourceId  -PlatformUpdateDomain 2;
```

Det här kommandot tvingar uppdatering av tjänsten att gå på UD 2 för den virtuella datorskala som anges av resurs-ID.
