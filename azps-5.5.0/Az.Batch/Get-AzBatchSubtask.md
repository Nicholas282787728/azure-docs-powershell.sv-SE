# Get-AzBatchSubtask

## SYNOPSIS
Hämtar underaktivitetsinformationen för den angivna aktiviteten.

## SYNTAX

### ODataFilter (standard)
```
Get-AzBatchSubtask [-JobId] <String> [-TaskId] <String> [-MaxCount <Int32>] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### ParentObject
```
Get-AzBatchSubtask [[-Task] <PSCloudTask>] [-MaxCount <Int32>] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Get-AzBatchSubtask** hämtar underaktivitetsinformationen för den angivna aktiviteten.
Underaktiviteter ger parallell bearbetning för enskilda aktiviteter och möjliggör exakt övervakning av aktivitetskörningen och -förlopp.

## EXEMPEL

### Exempel 1: Returnera alla underaktiviteter för en angiven aktivitet
```
PS C:\>$Context = Get-AzBatchAccountKey -AccountName "contosobatchaccount"
PS C:\> Get-AzBatchSubtask -JobId "Job-01" -TaskID "myTask" -BatchContext $Context
```

Med de här kommandona returneras alla underaktiviteter för uppgiften med ID myTask.
Det gör du genom att det första kommandot i exemplet skapar en objektreferens till kontonycklarna för batchkontot contosobatchaccount.
Den här objektreferensen lagras i en variabel med namnet $context.
Det andra kommandot använder sedan objektreferensen och cmdleten **Get-AzBatchSubtask** för att returnera alla underaktiviteter för myTask, en aktivitet som körs som en del av job-01.

Om du vill använda autentisering med delad nyckel i Get-AzBatchAccountKey använder du cmdleten för att få ett BatchAccountContext-objekt med dess snabbtangenter ifyllda.
