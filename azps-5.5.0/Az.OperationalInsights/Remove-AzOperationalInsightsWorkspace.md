# Remove-AzOperationalInsightsWorkspace

## SYNOPSIS
Tar bort en arbetsyta.

## SYNTAX

```
Remove-AzOperationalInsightsWorkspace [-ResourceGroupName] <String> [-Name] <String> [-ForceDelete] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESKRIVNING
Cmdleten **Remove-AzOperationalInsightsWorkspace** tar bort en befintlig arbetsyta.
Om den här arbetsytan var länkad till ett befintligt konto via *CustomerId-parametern* när det ursprungliga kontot skapades tas inte det bort i Driftsinformation-portalen.

## EXEMPEL

### Exempel 1: Ta bort en arbetsyta efter namn
```
PS C:\>Remove-AzOperationalInsightsWorkspace -ResourceGroupName "ContosResourceGroup" -Name "MyWorkspace"
```

Med det här kommandot tas arbetsytan MyWorkspace bort från resursgruppen ContosoResourceGroup.

### Exempel 2: Ta bort en arbetsyta med hjälp av pipelinen och utan bekräftelse
```
PS C:\>Get-AzOperationalInsightsWorkspace -ResourceGroupName "ContosResourceGroup" -Name "MyWorkspace" | Remove-AzOperationalInsightsWorkspace -Force
```

Det här kommandot använder cmdleten Get-AzOperationalInsightsWorkspace för att få arbetsytan MyWorkspace och skickar den sedan till cmdleten **Remove-AzOperationalInsightsWorkspace** genom att ta bort den med rörledningsoperatorn.
Eftersom *force-parametern* har angetts uppmanas du inte att göra det innan arbetsytan tas bort.

### Exempel 3: Tvinga borttagning av arbetsyta (kan inte återställas)
```
PS C:\> $workspace = New-AzOperationalInsightsWorkspace -ResourceGroupName $rgname -Name $wsname -Location $wslocation
PS C:\> $workspace | Remove-AzOperationalInsightsWorkspace -ForceDelete
```

Tvinga borttagning av en arbetsyta.
