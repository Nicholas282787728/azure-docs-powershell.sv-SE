# New-AzSupportTicket

## SYNOPSIS
Skapar ett support ärende.

## SYNTAX

## BESKRIVNING
Den här cmdleten kan användas för att skapa ett supportärenden för fakturering, prenumerationshantering, kvot eller tekniska problem. Använd Get-AzSupportService- Get-AzSupportProblemClassification-cmdlets för att identifiera Azure-tjänsten och dess motsvarande problemklassificering som du vill få support för. Du måste ange följande parametrar: 

    • Title
    • Description
    • Severity level
    • ProblemClassificationId
    • CustomerContactDetail (or individual customer contact parameters)

Du kan använda New-AzSupportContactProfileObject-cmdleten CustomerContactDetail-objekt.

Molnlösningsleverantörer kan skapa ett supportärende för kundens prenumerationer genom att logga in på kundens klientorganisation och ange deras klientorganisations-ID med parametern *CSPHomeTenantId.*

__För tekniska biljetter:__

Om du vill ange resursnamnet anger du ARM resursens resurs-ID med *parametern TechnicalTicketResourceId.* Se ett exempel nedan. 

__För kvotärenden:__

Om du vill begära en kvotökning för **Compute VM Cores**, **Batch,** **SQL Database** och SQL **Data Warehouse** tillhandahåller du ytterligare information under *Objektet QuotaTicketDetail.* QuotaTicketDetail-objektet består av 3 egenskaper enligt beskrivningen nedan. För detaljerad dokumentation, klicka [här.](https://aka.ms/supportrpquotarequestpayload)

    • QuotaChangeRequestSubType

        This is required for certain quota types when there is a sub type that you are requesting quota increase for. Example: Batch, SQL Database and SQL Data Warehouse have a sub type.

    • QuotaChangeRequestVersion

        This is required and indicates the version of the quota change request payload.

    • QuotaChangeRequests

        This is required and is a list of PSQuotaChangeRequest objects. PSQuotaChangeRequest object has 2 required properties.

        ○ Region

            This is the Azure location or region for which you are requesting quota increase. This is the Location property of Get-AzLocation cmdlet.
        
        ○ Payload

            This is where you specify the new limits for the selected quota type.


För detaljerad dokumentation om hur du skapar nyttolast för olika kvottyper, klicka [här](https://aka.ms/supportrpquotarequestpayload)
