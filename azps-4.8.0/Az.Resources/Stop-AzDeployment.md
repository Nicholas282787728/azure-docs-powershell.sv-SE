# Stop-AzDeployment

## Sammanfattning
Avbryta en pågående installation

## FRÅGESYNTAXEN

### StopByDeploymentName (standard)
```
Stop-AzDeployment [-Name] <String> [-PassThru] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzDeployment -Id <String> [-PassThru] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzDeployment -InputObject <PSDeployment> [-PassThru] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## PROBLEMBESKRIVNING
Cmdleten **Stop-AzDeployment** avbryter en distribution vid en prenumeration som har startat men inte slutförts.
För att stoppa en distribution måste distributionen ha ett ofullständigt etablerings tillstånd, till exempel etablering och inte ett slutfört tillstånd, såsom etablerad eller misslyckad.

Om du vill skapa en distributions prenumerations omfattning använder du New-AzDeployment cmdlet.

Denna cmdlet stoppar bara en drift sättning. Använd parametern *namn* för att stoppa en specifik distribution.

## BESKRIVS

### Exempel 1
```
PS C:\>Stop-AzDeployment -Name "deployment01"
```

Det här kommandot avbryter en drift sättning "deployment01" för det aktuella abonnemanget.

### Exempel 2
```
PS C:\>Get-AzDeployment -Name "deployment01" | Stop-AzDeployment
```

Det här kommandot får distributionen "deployment01" vid den aktuella prenumerations omfattningen och säger upp det.
