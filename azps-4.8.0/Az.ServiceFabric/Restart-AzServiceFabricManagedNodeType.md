# Restart-AzServiceFabricManagedNodeType

## Sammanfattning
Starta om specifika noder från nodtypen.

## FRÅGESYNTAXEN

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## PROBLEMBESKRIVNING
Starta om specifika noder från nodtypen. Den inaktive ras av tjänstens Fabric-noder innan de startas om och du aktiverade dem igen när de kommer tillbaka. Om det sker på primära nodtyper kan det ta en stund eftersom det inte går att starta om alla noder samtidigt. Use-ForceRestart tvinga åtgärden även om Service Fabric inte kan inaktivera noderna men använda försiktighet eftersom det kan orsaka data förlust om tillstånds krävande arbets belastningar körs på noden.

## BESKRIVS

### Exempel 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Starta om nod 0 och 3 på nodtypen.
