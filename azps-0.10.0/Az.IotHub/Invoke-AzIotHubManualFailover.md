# Invoke-AzIotHubManualFailover

## Sammanfattning
Aktivera redundans för IoT Hub till det geobaserade återställnings området för katastrof återställning.

## FRÅGESYNTAXEN

### ResourceSet (standard)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## PROBLEMBESKRIVNING
Den initierar din IoT-hubb till den sekundära platsen. Med den här åtgärden kan du förlora tid och telemetri för din lösning. Det kan ta flera minuter att slutföra. Var försiktig när du använder den.

## BESKRIVS

### Exempel 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Initierar redundans för "myiothub" IoT Hub.

### Exempel 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Initierar redundansväxlingen av "myiothub" IoT Hub i bakgrunden.
