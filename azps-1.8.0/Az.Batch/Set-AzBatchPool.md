# Set-AzBatchPool

## Sammanfattning
Uppdaterar egenskaperna för en pool.

## FRÅGESYNTAXEN

```
Set-AzBatchPool [-Pool] <PSCloudPool> -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## PROBLEMBESKRIVNING
Cmdleten **set-AzBatchPool** uppdaterar egenskaperna för en pool i Azure Batch-tjänsten.
Använd Get-AzBatchPool cmdlet för att hämta ett **PSCloudPool** -objekt.
Ändra egenskaperna för objektet och Använd sedan aktuell cmdlet för att bekräfta ändringarna i kommando tjänsten.

## BESKRIVS

### Exempel 1: uppdatera en pool
```
PS C:\>$Pool = Get-AzBatchPool "ContosoPool" -BatchContext $Context
PS C:\> $StartTask = New-Object Microsoft.Azure.Commands.Batch.Models.PSStartTask
PS C:\> $StartTask.CommandLine = "cmd /c echo example"
PS C:\> $Pool.StartTask = $StartTask
PS C:\> Set-AzBatchPool -Pool $Pool -BatchContext $Context
```

Det första kommandot får en pool genom att använda **Get-AzBatchPool** och lagrar den sedan i $pool variabel.
Nästa tre kommandon ändrar specifikationen för start uppgiften för $Pool-objektet.
Det sista kommandot uppdaterar kommando tjänsten för att matcha det lokala objektet i $Pool.

## MALLPARAMETRAR

### -BatchContext
Anger den **BatchAccountContext** -instans som denna cmdlet använder för att interagera med kommando tjänsten.
Om du använder Get-AzBatchAccount cmdlet för att hämta din BatchAccountContext används Azure Active Directory-identifiering när den fungerar tillsammans med kommando tjänsten. Om du vill använda delad nyckel-identifiering kan du i stället använda Get-AzBatchAccountKeys cmdlet för att hämta ett BatchAccountContext-objekt med dess åtkomst nycklar ifyllda. När du använder autentisering med delad nycklar används den primära åtkomst tangenten som standard. Om du vill ändra den nyckel som ska användas ställer du in egenskapen BatchAccountContext.
