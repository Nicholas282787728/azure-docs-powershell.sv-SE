# Initialize-AzRecoveryServicesBackupProtectableItem

## Sammanfattning
Det här kommandot utlöser identifieringen av oskyddade objekt av den angivna typen av arbets belastning i den angivna behållaren. Om DB-programmet inte är skyddat automatiskt använder du det här kommandot för att identifiera nya DBs när de läggs till och fortsätter för att skydda dem.

## FRÅGESYNTAXEN

```
Initialize-AzRecoveryServicesBackupProtectableItem [-Container] <ContainerBase> [-WorkloadType] <WorkloadType>
 [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## PROBLEMBESKRIVNING
cmdleten söker efter specifika arbets belastningar i en behållare. Då utlöses en åtgärd som skapar skyddade objekt.

## BESKRIVS

### Exempel 1
```
PS C:\> Initialize-AzRecoveryServicesProtectableItem -Container $Container –WorkloadType “MSSQL”
```

Cmdleten kör en identifierings åtgärd för nya oskyddade objekt.
