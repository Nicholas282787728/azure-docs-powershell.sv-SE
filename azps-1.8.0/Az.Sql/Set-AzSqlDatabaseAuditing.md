## PROBLEMBESKRIVNING
Cmdleten **set-AzSqlDatabaseAuditing** ändrar gransknings inställningarna för en Azure SQL-databas.
Använd cmdleten *ResourceGroupName* , *servername* och *databasename* för att identifiera databasen.
Mål platsen för gransknings loggar bestäms genom att ange en av följande växel parametrar: BlobStorage, LogAnalytics eller EventHub (om ingen är angiven är standardvärdet BlobStorage).
Använd parametern *State* för att aktivera/inaktivera principen.
När mål för gransknings loggar är en blob-lagring anger du parametern *StorageAccountName* för att fastställa lagrings konto för gransknings loggar och parametern *StorageKeyType* för att definiera lagrings nycklarna. Du kan också definiera bevarande för gransknings loggar genom att ange värdet för *RetentionInDays* -parametern för att definiera perioden för gransknings loggar.
Om cmdleten lyckas och du använder *Passthru* -parametern returneras ett objekt som beskriver de aktuella inställningarna för granskning, utöver databasens identifierare.
Databas identifierare inkluderar, men är inte begränsade till, **ResourceGroupName** , **servername** och **databasename**.

## BESKRIVS

### Exempel 1: Aktivera gransknings princip för blob-lagring i en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01"
```

### Exempel 2: inaktivera gransknings princip för blob-lagring för en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

### Exempel 3: Aktivera gransknings princip för blob-lagring för en Azure SQL-databas med ett lagrings konto från ett annat abonnemang
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -StorageAccountSubscriptionId "7fe3301d-31d3-4668-af5e-211a890ba6e3"
```

### Exempel 4: Aktivera gransknings princip för blob-lagring för en Azure SQL-databas med avancerad filtrering med ett T-SQL-predikat
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01" -PredicateExpression "statement <> 'select 1'"
```

### Exempel 5: ta bort inställningen Advanced filter from gransknings princip för blob-lagring i en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01" -PredicateExpression ""
```

### Exempel 6: Aktivera gransknings principen för händelsehubben för en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -EventHub -EventHubName "EventHubName" -EventHubAuthorizationRuleResourceId "EventHubAuthorizationRuleResourceId"
```

### Exempel 7: inaktivera gransknings princip för händelsehubben för en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -EventHub
```

### Exempel 8: Aktivera gransknings princip för logganalys för en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -LogAnalytics -WorkspaceResourceId "/subscriptions/4b9e8510-67ab-4e9a-95a9-e2f1e570ea9c/resourceGroups/insights-integration/providers/Microsoft.OperationalInsights/workspaces/viruela2"
```

### Exempel 9: inaktivera gransknings principen för logganalys för en Azure SQL-databas
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -LogAnalytics
```

### Exempel 10: inaktivera, genom pipeline, gransknings princip för logganalys för en Azure SQL-databas
```
PS C:\>Get-AzSqlDatabase -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" | Set-AzSqlDatabaseAuditing -LogAnalytics -State Disabled
```

### -AuditActionGroup
Den rekommenderade uppsättningen åtgärds grupper som ska användas är följande kombination – detta granskar alla frågor och lagrade procedurer som körs mot databasen samt lyckade och misslyckade inloggningar:  
  
"BATCH_COMPLETED_GROUP",  
"SUCCESSFUL_DATABASE_AUTHENTICATION_GROUP",  
"FAILED_DATABASE_AUTHENTICATION_GROUP"  
Ovanstående kombination är även den uppsättning som är konfigurerad som standard. Dessa grupper täcker alla SQL-uttryck och lagrade procedurer som körs mot databasen och ska inte användas i kombination med andra grupper eftersom detta kommer att leda till dubbletter av gransknings loggar.
Mer information finns i https://docs.microsoft.com/en-us/sql/relational-databases/security/auditing/sql-server-audit-action-groups-and-actions#database-level-audit-action-groups .
