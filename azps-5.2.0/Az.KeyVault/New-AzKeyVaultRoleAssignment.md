# New-AzKeyVaultRoleAssignment

## Sammanfattning
Tilldelar den angivna RBAC-rollen till den angivna huvud delen, vid det angivna omfånget.

## FRÅGESYNTAXEN

 -SignInName <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]

## PROBLEMBESKRIVNING
Använd `New-AzKeyVaultRoleAssignment` kommandot för att ge åtkomst.
Åtkomst beviljas genom att den lämpliga RBAC-rollen tilldelas till dem vid rätt scope.
Uppgiftens ämne måste anges.
Ange en användare med hjälp av SignInName eller Azure AD ObjectId-parametrar.
Om du vill ange en säkerhets grupp använder du Azure AD ObjectId-parameter.
Om du vill ange ett Azure AD-program använder du ApplicationId-eller ObjectId-parametrar.
Rollen som tilldelas måste anges med RoleDefinitionName PR RoleDefinitionId-parametern. Omfattningen där åtkomst ges kan anges. Det är standardinställningen för det valda abonnemanget.

## BESKRIVS

### Exempel 1
```powershell
PS C:\> New-AzKeyVaultRoleAssignment -HsmName myHsm -RoleDefinitionName "Managed HSM Policy Administrator" -SignInName user1@microsoft.com

RoleDefinitionName               DisplayName                    ObjectType Scope
------------------               -----------                    ---------- -----
Managed HSM Policy Administrator User 1 (user1@microsoft.com)   User       /
```

I det här exemplet tilldelas rollen "hanterad HSM-princip administratör" till användaren " user1@microsoft.com " högst upp.
