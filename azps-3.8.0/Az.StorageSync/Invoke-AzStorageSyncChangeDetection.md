# Invoke-AzStorageSyncChangeDetection

## Sammanfattning
Det här kommandot kan användas för att manuellt identifiera ändringar av namn områden. Den kan riktas till hela resursen, undermappen eller uppsättningen filer. Högst 10 000 objekt kan identifieras. Om omfattningen av ändringar är känd för dig kan du begränsa körningen av det här kommandot till delar av namn området så att ändrings kontrollen kan slutföra snabbt och inom objekt gränsen för 10 000.

> [!Note]  
> Invoke-AzStorageSyncChangeDetection cmdlet identifierar inte filer som tas bort i Azure-fildelningen. Om filer tas bort i Azure File Share identifieras de när [ändrings kontroll jobbet](https://docs.microsoft.com/azure/storage/files/storage-sync-files-troubleshoot?tabs=portal1%2Cazure-portal#afs-change-detection) körs.

## PROBLEMBESKRIVNING
Med jämna mellanrum kontrollerar Azure File Sync namn området i en synkroniserad Azure File-resurs för ändringar som kommer till fil resursen på annat sätt än synkronisering. Målet är att identifiera dessa ändringar och sedan synkronisera dem till anslutna servrar. Det här kommandot kan användas för att manuellt identifiera ändringar av namn områden. Den kan riktas till hela resursen, undermappen eller uppsättningen filer. Om omfattningen av ändringar är känd för dig kan du begränsa körningen av det här kommandot till delar av namn området så att ändrings kontrollen kan slutföras snabbt och i en 10 000-ändrings gräns.

## BESKRIVS

### Exempel 1
```powershell
PS C:\> Invoke-AzStorageSyncChangeDetection -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -CloudEndpointName "myCloudEndpointName" -Path "Data","Reporting\Templates"
```

I det här exemplet körs ändrings kontroll i katalogerna "data" och "Reporting\Templates" för en synkroniserad Azure-fildelning. Alla sökvägar är relativa till roten för Azure File Share-namnområdet.

### Exempel 2
```powershell
PS C:\> Invoke-AzStorageSyncChangeDetection -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -CloudEndpointName "myCloudEndpointName" -Path "Data\results.xslx","Reporting\Templates\generated.pptx"
```

I det här exemplet körs ändrings kontroll för en uppsättning filer som är kända för att den som anropas har ändrats. Målet är att Azure-filsynkroniseringen ska upptäcka och synkronisera dessa ändringar.

### Exempel 3
```powershell
PS C:\> Invoke-AzStorageSyncChangeDetection -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -CloudEndpointName "myCloudEndpointName" -DirectoryPath "Examples" -Recursive
```

I det här exemplet körs ändrings spårning för "exempel"-katalogen och identifierar ändringar i under katalogerna rekursivt. Kom ihåg att det här kommandot kan upptäcka upp till 10 000-ändringar innan det avbryts automatiskt. Om du misstänker att fler än 10 000 ändringar har inträffat kör du kommandot på under delarna i namn området.
